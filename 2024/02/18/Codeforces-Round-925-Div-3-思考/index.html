Codeforces Round 925 (Div. 3)思考

\(\Delta Rating : +152\)

这也是我写的第一个比赛，感觉还行，还加了100多分，目标绿名。

A 题

就是将a ~ z映射成1到26，给你一个数字，让你构造出字典序最小的拥有三个字母的单词，并且通过映射，三个字母相加等于那个数字。

也仅仅是每个字母位枚举，判断一下就 OK，没什么难的， 最多 \(26^3\) 次找到。

B 题

给你一串数字，只能从1到n，只能从前面的数字移走一部分给后面的数字，让你计算能不能将所有数字给变成相同的。

思路简单：先算出平均数（如果全部数字都相同，那么一定等于平均数），那么从1到n计算出它与平均分差的东西，如果小于0，那么说明一定不行，否则行。

注意一下有数组大小 \(2 \times 10^5\)，前几次开小了，罚时了3次，哭死。

C 题

给你一串数字，你能做一种操作（且只能做 0 次或 1 次），你可以选择一个区间，让它全部变成你所给的数字，所付出的代价是 \(j - i + 1\)。输出让全部数字变成一样的最小的代价。

其实观察样例就知道，从两边开始去做一次操作是最优的，用类似于双指针的东西。 * 先判断两边是否相等，如果相等，那么直接找到两边不相等的地方来做操作 * 如果不相等，看两边哪一边相等的更多一些，那边多就从那个边的不相等的地方开始做操作。

注意一些数组得清零，不然就会多 WA 一次。

D 题

这道题比赛时，没做出来，还是数学基本功差了，同时也与晚上太晚无法清醒思考有关。

给你一个数组，让你找到数组中最美丽的数对的个数。

一个 \((i,j)\) 认为它是美丽的当且仅当：（\(x\) 与 \(y\) 给定） * \(a_i + a_j\) 能够整除于 \(x\) * \(a_i - a_j\) 能够整除于 \(y\)

首先，第一条能推出 \((a_i \bmod x + a_j \bmod x)\bmod x = 0\)，所以 \((a_i \bmod x + a_j \bmod x)\) 等于 \(0\) 或 \(x\)。

又因为 \(0 = x \bmod x\)，所以 \(a_i \bmod x=(x − a_j \bmod x)\bmod x\)

因为 \((a_i - a_j)\mod y = 0\) ，能推出 \(a_i \bmod y = a_j \bmod y\)

通过上面的两个式子就可以快速推出每对美丽的数对。最后通过 Map 来存放每个数对。

注意：不开Long Long见祖宗
